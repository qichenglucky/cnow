# PRD 标准版

## 1. 背景
新服务上线往往需要跨越仓库、CI/CD、域名、日志、监控、审批、回滚等多个系统，流程割裂、配置重复、上线耗时长。

## 2. 产品目标
1. 让用户通过最少信息创建一个服务
2. 自动生成仓库、CI、测试环境、域名、日志、监控和生产发布配置
3. 统一发布过程的可视化、可审计和可回滚能力
4. 用 AI 降低配置门槛并增强风险分析能力

## 3. MVP 范围
### 必做
- 服务创建
- 仓库创建或接入
- CI 配置生成
- 测试环境部署
- 泛域名配置
- SLS 日志接入
- 基础监控接入
- 生产发布
- 发布历史与流程追踪
- AI 方案生成与风险分析

### 暂不做
- 自动修复执行
- 复杂多云编排
- 自定义插件市场
- 高级工作流编排器

## 4. 角色与权限
- 平台管理员：模板、域名、审批流、权限、规范
- 服务负责人：创建、发布、回滚、查看全量观测
- 开发者：查看、测试环境发布、查看日志监控、发起生产申请
- 观察者：只读

## 5. 核心流程
### 新建服务
表单或自然语言输入 -> AI 生成方案 -> 用户确认 -> 创建仓库/CI/测试环境 -> 接入域名/日志/监控 -> 进入服务详情页。

### 测试环境发布
选择分支或 commit -> 构建 -> 部署测试环境 -> 分配测试域名 -> 接入日志和监控 -> 健康检查。

### 生产发布
选择版本 -> 风险分析 -> 审批 -> 灰度或全量发布 -> 健康观察 -> 失败则回滚 -> 生成审计记录。

## 6. 页面范围
- 首页
- 服务详情页
- 发布页
- 日志页
- 监控页
- AI 侧栏

## 7. 验收标准
- 能创建服务
- 能生成 CI
- 能部署测试环境
- 能访问测试域名
- 能发生产
- 能审批
- 能回滚
- 能追踪历史
