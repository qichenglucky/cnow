# 步骤存档索引

## Step 01 - 项目定义
- 确定项目名：即码 CodeNow.ai / Cnow
- 确定产品定位：AI 原生集成发布平台
- 确定核心闭环：创建、发布、监控、日志、回滚

## Step 02 - PRD 输出
- 完成 PRD 标准版
- 完成评审修订稿
- 完成页面交互原型说明
- 完成测试验收用例表

## Step 03 - 架构决策
- 选型 Temporal / Kafka / PostgreSQL / K8s / Go / React
- 决定主键使用自增 bigint
- 决定不预留多租户字段

## Step 04 - 设计落地
- 完成技术架构设计
- 完成概要设计
- 完成详细设计

## Step 05 - 工作流与异常规范
- 完成发布工作流设计文档
- 完成接口错误码与异常处理规范

## Step 06 - 当前状态
- 所有前置文档已归档到本项目
- 后续可进入接口实现、数据库建表、工作流编码和前端开发

## Step 07 - 实施文档补齐
- 完成接口定义文档
- 完成数据库表设计文档
- 完成发布工作流伪代码
- 更新 README 文档索引

## Step 08 - 架构底线约束
- 将可用性、性能、可维护性、安全性、可观测性、交付效率、成本控制纳入架构硬约束
- 补充架构设计与实现约束文档
- 更新项目索引

## Step 09 - 实施骨架启动
- 建后端项目结构
- 建数据库迁移脚本
- 建工作流引擎接入层
- 建前端页面骨架
